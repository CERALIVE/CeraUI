## Design Context

### Users

Live streamers, broadcasters, and content creators — from professional broadcast engineers to passionate field operators — configuring a CeraLive streaming encoder device. Primary context: field broadcasts under time pressure, event management on tablets, studio setup at desktop. They configure multi-link bonded network connections (cellular modems, Wi-Fi, Ethernet), streaming pipelines, video/audio encoding, and server targets. During a live broadcast they need at-a-glance telemetry and immediate control, not configuration menus.

### Brand Personality

**Ground Control** — precision telemetry instrument. A satellite operations center. Warm near-black graphite surfaces, phosphor-lime live indicators, amber/coral state signals. Confident, instrument-calibrated, field-ready.

Three words: **Calibrated. Live. Controlled.**

Emotional goal: the operator feels the stream is in good hands — every indicator reads at a glance, every action is unambiguous, the interface never surprises them at a critical moment.

### Aesthetic Direction

**Instrument-calibrated, dark-first** — the precision of satellite ground control meets the immediacy of live telemetry. Warm graphite hue 150, phosphor lime primary. Every element earns its place.

- **Tone**: Dark and purposeful. The base experience is dark graphite — the device lives in controlled environments, events, and broadcast settings where dark reduces eye strain and the phosphor-lime accents pop. Light mode is first-class but not the hero.
- **Color**: OKLCH-based token system. Dark base `oklch(0.15 0.012 150)` — warm near-black graphite, hue 150. Near-white light mode. Phosphor lime `oklch(0.86 0.2 128)` dark / `oklch(0.4 0.18 128)` light is the primary accent — the "live" signal color. Amber standby `oklch(0.8 0.16 75)`. Coral error `oklch(0.64 0.2 25)`. Spectral `--link-1..6` ramp for per-link signal identity.
- **Typography**: Space Grotesk Variable (sans, geometric warmth for headings and UI labels). JetBrains Mono Variable (mono, for data values, bitrates, telemetry, code). System fallbacks for CJK and Arabic scripts (Space Grotesk lacks those glyphs). Both served locally.
- **Anti-references**: Glassmorphism as decoration (no backdrop-blur cards by default). Gradient text (background-clip: text — never). Side-stripe accent borders on cards. Generic SaaS dashboard patterns (hero-metric template, identical card grids, per-section uppercase eyebrow labels). Warm cream/beige/sand backgrounds. The previous CeraUI identity (cool-hued neutrals, teal/turquoise primary, deep cool-dark background).
- **References**: The instrument clarity of satellite operations software, the data density of broadcast monitoring interfaces, the restraint of Blackmagic Design — adapted for a hardware configuration context.
- **Theme**: Both dark and light modes, both first-class. Dark is the hero — warm near-black graphite with phosphor-lime live indicators.
- **Responsive**: Must excel across mobile (field use), tablet (event management), and desktop (studio configuration). 1024×600 kiosk viewport is a real target (touchscreen attachment). Not "shrink the desktop" — genuinely adapted layouts per context.

### Design Principles

1. **Instrument Clarity**: Every surface is a display, not a form. Status is primary; configuration is contextual (accessed via focused dialogs). The operator should know the stream state at a glance without opening anything.
2. **Dark-First Hero**: The base experience is dark graphite. Phosphor-lime accents pop against it — the "live" signal reads immediately. Light mode is designed with equal care but is not the default register.
3. **Dialog-Driven Configuration**: Configuration complexity lives in focused, contextual dialogs and sheets — not inline mega-forms. Each dialog owns one concern. The three destinations (Live, Network, Settings) are scannable, not scroll-heavy forms.
4. **Live-Data Discipline**: Real-time telemetry (bitrate, signals, sensors) is throttled, smoothed, and staleness-aware. Live values must degrade gracefully on disconnect — never show misleading fresh-looking data when the connection is down.
5. **Touch-Inclusive by Default**: Hit targets, spacing, and layout must remain functional when a touchscreen is attached. 44px minimum touch targets. The foundation is web-first but kiosk-ready.
